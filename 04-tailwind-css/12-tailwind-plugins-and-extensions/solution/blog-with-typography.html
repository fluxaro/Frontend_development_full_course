Understanding JavaScript Closures: A Complete Guide

Closures are one of the most powerful and often misunderstood concepts in JavaScript. Master them and you'll write better, more expressive code.

AR

Alex Rivera

May 11, 2026

What is a Closure?

A closure is a function that has access to variables from its outer (enclosing) scope, even after the outer function has finished executing. This is possible because JavaScript functions form a closure over the variables in their surrounding scope at the time they are created.

When you create a function inside another function, the inner function "closes over" the outer function's variables. Those variables remain accessible to the inner function even after the outer function returns.

A Simple Example

The classic counter example demonstrates closures clearly:

function makeCounter() {
  let count = 0; // this variable is "closed over"

  return function() {
    count++;
    return count;
  };
}

const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

The inner function remembers count even though makeCounter has already returned. Each call to counter() increments the same count variable.

Closures are not a special syntax — they are a natural consequence of how JavaScript handles scope and function creation.

Practical Uses of Closures

Closures are used everywhere in JavaScript. Here are the most common patterns:

1. Data Privacy and Encapsulation

Closures let you create private variables that can't be accessed from outside:

function createBankAccount(initialBalance) {
  let balance = initialBalance; // private

  return {
    deposit(amount) { balance += amount; },
    withdraw(amount) { balance -= amount; },
    getBalance() { return balance; }
  };
}

const account = createBankAccount(100);
account.deposit(50);
console.log(account.getBalance()); // 150
// balance is not directly accessible

2. Factory Functions

Create multiple instances with their own private state:

function createMultiplier(factor) {
  return (number) => number * factor;
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(5));  // 10
console.log(triple(5));  // 15

Common Pitfalls

The most common closure mistake involves loops:

// Bug: all buttons alert "3"
for (var i = 0; i < 3; i++) {
  buttons[i].onclick = function() { alert(i); };
}

// Fix: use let (block-scoped)
for (let i = 0; i < 3; i++) {
  buttons[i].onclick = function() { alert(i); };
}

Key Takeaways

  • A closure is created every time a function is created inside another function
  • The inner function retains access to the outer function's variables
  • Closures enable data privacy, factory functions, and stateful callbacks
  • Use let instead of var in loops to avoid closure bugs
