What is Tailwind CSS?

A utility-first CSS framework — style directly in HTML with small, single-purpose classes

Traditional CSS vs Tailwind

Traditional CSS

/* style.css */
.card {
  background: white;
  border-radius: 8px;
  padding: 1.5rem;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}

.card-title {
  font-size: 1.25rem;
  font-weight: 700;
  margin-bottom: 0.5rem;
}

.card-text {
  color: #666;
  line-height: 1.6;
}
<!-- index.html -->
<div class="card">
  <h3 class="card-title">Title</h3>
  <p class="card-text">Text</p>
</div>

Tailwind CSS ✅

<!-- No separate CSS file needed -->
<div class="bg-white rounded-xl p-6
            shadow-md">
  <h3 class="text-xl font-bold mb-2">
    Title
  </h3>
  <p class="text-gray-500 leading-relaxed">
    Text
  </p>
</div>

No CSS file. No class naming. No context switching.

CLI / Vite Recommended

Full features, tree-shaking, production-ready.
