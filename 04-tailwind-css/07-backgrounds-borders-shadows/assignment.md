# Assignment — Card Showcase

## What You Are Building

A showcase page demonstrating mastery of Tailwind's background, border, shadow, and visual effect utilities — through a collection of polished card variants.

---

## Requirements

Create `card-showcase.html` with the Tailwind CDN.

### Card Variants (build all of these)

- [ ] **Default card** — white background, `border border-gray-100`, `shadow-sm`, `rounded-2xl`
- [ ] **Elevated card** — white background, `shadow-xl`, no border, `rounded-2xl`
- [ ] **Outlined card** — transparent background, `border-2 border-blue-500`, `rounded-2xl`
- [ ] **Gradient card** — `bg-gradient-to-br` with two colors, white text, `rounded-2xl`
- [ ] **Glassmorphism card** — placed over a gradient background, uses `backdrop-blur-md bg-white/10 border border-white/20`
- [ ] **Colored shadow card** — uses `shadow-lg shadow-{color}/50` with a matching background color
- [ ] **Image card** — uses a gradient as a simulated background image with `bg-cover bg-center`, text overlay with `bg-black/50`
- [ ] **Dashed border card** — uses `border-2 border-dashed border-gray-300` (upload/add new style)

### Each Card Must Have
- [ ] A title
- [ ] A short description
- [ ] A button or link at the bottom

### Layout
- [ ] Cards displayed in a responsive grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`
- [ ] Consistent `gap-6` between cards
- [ ] A page header with a gradient background

---

## What Good Looks Like

- Each card is visually distinct and demonstrates its specific technique
- The glassmorphism card is placed over a colorful background so the blur effect is visible
- Colored shadows match the card's background color
- The page looks like a real UI component library showcase

---

## Submission

Submit `card-showcase.html`. Open it in a browser and confirm each card variant is clearly different from the others.
