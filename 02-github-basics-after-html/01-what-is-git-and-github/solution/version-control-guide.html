What Is Version Control?

Version control is a system that records changes to files over time so you can recall specific versions later.

Without Version Control

Imagine you are writing an essay. You save it as:

  • essay.docx
  • essay-final.docx
  • essay-final-v2.docx
  • essay-ACTUALLY-final.docx
  • essay-FINAL-for-real.docx

This is messy, confusing, and wastes space. Version control solves this problem.

With Version Control

You have one file. Git tracks every change with a timestamp and message. You can go back to any version instantly.

What Is Git?

Git is a distributed version control system created by Linus Torvalds in 2005. It runs on your computer and tracks changes to your files.

What Git Does

  • Tracks every change to every file in your project
  • Lets you create snapshots (commits) of your work
  • Lets you create branches to work on features separately
  • Lets you merge branches back together
  • Works completely offline

Basic Git Commands

git init              # Start tracking a folder with Git
git add .             # Stage all changes
git commit -m "msg"   # Save a snapshot with a message
git log               # View commit history
git status            # See what has changed

What Is GitHub?

GitHub is a web-based platform that hosts Git repositories. It is owned by Microsoft and launched in 2008.

What GitHub Adds

  • Cloud storage for your Git repositories
  • A web interface to browse your code and history
  • Pull Requests for code review and collaboration
  • Issues for tracking bugs and feature requests
  • GitHub Actions for automated testing and deployment
  • GitHub Pages for free website hosting

GitHub Alternatives

  • GitLab — popular for self-hosting and CI/CD
  • Bitbucket — popular with Atlassian tools (Jira)
  • Azure DevOps — Microsoft's enterprise platform

Glossary of Key Terms

Repository (repo)
A project folder tracked by Git. Contains all files and the complete history of changes.
Commit
A saved snapshot of your project at a specific moment. Has a unique ID and a message.
Branch
A parallel version of your code. Lets you work on features without affecting the main branch.
Main / Master
The default primary branch of a repository. Most teams use "main" today.
Merge
Combining the changes from one branch into another.
Conflict
When two branches change the same line of code differently. Must be resolved manually.
Remote
A version of your repository stored on another computer or server (like GitHub).
Push
Sending your local commits to the remote repository.
Pull
Downloading changes from the remote repository to your local machine.
Clone
Downloading a complete copy of a remote repository to your computer.
Fork
Creating your own copy of someone else's GitHub repository under your account.
Pull Request (PR)
A request to merge your branch into another branch. Used for code review.
Issue
A GitHub feature for tracking bugs, feature requests, and tasks.
