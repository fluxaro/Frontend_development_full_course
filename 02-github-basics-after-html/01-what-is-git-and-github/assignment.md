# Assignment — Git vs GitHub Explainer Page

## Overview

Build an HTML page that teaches someone what Git and GitHub are. Imagine you're explaining it to a friend who has never heard of either. Your page should be clear, visual, and actually useful.

**Due:** Next class  
**Submission:** Upload the file to your class folder or share the link if you've pushed it to GitHub

---

## Requirements

Your HTML page must include all of the following:

### 1. Page Title and Introduction
- A clear `<h1>` title
- A short intro paragraph (2–3 sentences) explaining what this page is about

### 2. Comparison Table — Git vs GitHub
Create an HTML `<table>` that compares Git and GitHub across at least 5 categories:
- What it is
- Where it runs
- Requires internet?
- Main purpose
- Created by

### 3. Key Terms Section
Define each of these terms in your own words using a **description list** (`<dl>`, `<dt>`, `<dd>`):
- Repository
- Commit
- Branch
- Merge
- Push
- Pull
- Clone
- Fork

### 4. Visual Timeline
Create a visual timeline using HTML and inline styles (or a `<table>`) showing how version control works. Show at least 4 steps:
1. You write code
2. You stage your changes
3. You commit (save a snapshot)
4. You push to GitHub

You can use emojis, arrows (→), or simple ASCII art to make it visual.

### 5. "Why Use Version Control?" Section
A list (`<ul>` or `<ol>`) of at least 5 reasons developers use version control. Write them in your own words — don't just copy from the README.

### 6. Common Misconceptions Section
Address at least 2 common misconceptions, for example:
- "Git and GitHub are the same thing"
- "You need GitHub to use Git"
