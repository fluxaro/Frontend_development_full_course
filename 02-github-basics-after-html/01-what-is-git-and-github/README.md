# 01 — What is Git and GitHub?

## Overview

Before we start pushing code to the internet, we need to understand the tools that make it possible: **Git** and **GitHub**. These are two different things that work together, and understanding the difference is the first step.

---

## The Video Game Analogy 🎮

Think about playing a video game. Every so often you hit **Save** — and the game records exactly where you are, what you have, and what you've done. If you mess up later, you can reload that save and try again.

**Git is like that save system — but for your code.**

Every time you make a meaningful change to your project, you create a "save point" (called a **commit**). If something breaks, you can go back to any previous save point. You can also create alternate timelines (called **branches**) to try new ideas without risking your main project.

---

## What is Git?

**Git** is a **version control system** that runs on your computer (locally).

- It tracks every change you make to your files over time
- It lets you go back to any previous version
- It lets you work on multiple versions of a project simultaneously (branches)
- It was created by Linus Torvalds (the same person who created Linux) in 2005
- It is **free and open source**
- It works entirely on your local machine — no internet required

---

## What is GitHub?

**GitHub** is a **website** (github.com) that hosts Git repositories in the cloud.

- It stores your Git repos online so you don't lose them
- It lets you share your code with others
- It enables collaboration — multiple developers can work on the same project
- It provides tools like Issues, Pull Requests, and Actions
- It is owned by Microsoft (acquired in 2018)
- It is **not** the only option — alternatives include GitLab and Bitbucket

---

## Git vs GitHub — The Key Difference

| | Git | GitHub |
|---|---|---|
| **What it is** | Software (a tool) | A website (a service) |
| **Where it lives** | Your computer | The internet |
| **Requires internet?** | No | Yes |
| **Main purpose** | Track changes locally | Host and share repos online |
| **Created by** | Linus Torvalds | Tom Preston-Werner & team |
| **Alternatives** | Mercurial, SVN | GitLab, Bitbucket, Gitea |

**Simple way to remember:** Git is the engine. GitHub is the garage where you park and show off your car.

---

## Why Do Developers Use Version Control?

1. **Safety net** — Never lose work. Every version is saved.
2. **History** — See exactly what changed, when, and who changed it.
3. **Collaboration** — Multiple people can work on the same codebase without overwriting each other.
4. **Experimentation** — Try new features in a branch without breaking the main project.
5. **Deployment** — Many hosting services deploy directly from GitHub.
6. **Portfolio** — GitHub is your coding resume. Employers look at it.

---

## Key Terms

| Term | Definition |
|---|---|
| **Repository (repo)** | A project folder tracked by Git |
| **Commit** | A saved snapshot of your changes |
| **Branch** | A parallel version of your project |
| **Merge** | Combining one branch into another |
| **Push** | Sending your local commits to GitHub |
| **Pull** | Getting the latest changes from GitHub to your local machine |
| **Clone** | Downloading a copy of a GitHub repo to your computer |
| **Fork** | Making your own copy of someone else's GitHub repo |
