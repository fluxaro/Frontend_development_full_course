Git vs GitHub — What Is the Difference?

Many beginners confuse Git and GitHub. They are related but different tools. This page explains both clearly.

What Is Git?

Git is a version control system. It runs on your computer and tracks changes to your files over time.

Think of Git like a save system in a video game. Every time you make progress, you can save a checkpoint. If something goes wrong, you can go back to any previous checkpoint.

  • Git is software you install on your computer
  • It works entirely offline
  • It tracks every change you make to your files
  • It lets you go back to any previous version
  • It lets you create branches to work on features separately

What Is GitHub?

GitHub is a website that hosts Git repositories in the cloud. It is like Google Drive, but specifically for code tracked with Git.

  • GitHub is a website (github.com)
  • It stores your Git repositories online
  • It lets you share your code with others
  • It enables collaboration on the same codebase
  • It provides tools like Issues, Pull Requests, and Actions

Key Git Terms

Repository (repo)
A folder tracked by Git. Contains all your files and the full history of changes.
Commit
A saved snapshot of your files at a specific point in time. Like a checkpoint.
Branch
A separate line of development. Lets you work on a feature without affecting the main code.
Merge
Combining changes from one branch into another.
Push
Uploading your local commits to a remote repository (like GitHub).
Pull
Downloading changes from a remote repository to your local machine.
Clone
Downloading a complete copy of a remote repository to your computer.
Fork
Creating your own copy of someone else's GitHub repository.

The Workflow

  1. You write code on your computer
  2. Git tracks the changes (local)
  3. You commit the changes with a message
  4. You push the commits to GitHub (remote)
  5. Your team can pull your changes and collaborate
