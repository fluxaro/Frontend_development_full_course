# Class Work — Create and Explore a GitHub Repository

## Goal

Create a real GitHub repository, explore every part of the interface, and understand what each section does.

**Time:** 20–25 minutes

---

## Part 1 — Create a New Repository

1. Go to https://github.com and log in
2. Click the **+** icon → **New repository**
3. Fill in the form:
   - **Name:** `bootcamp-practice` (or any name you like)
   - **Description:** "Practice repository for learning GitHub"
   - **Visibility:** Public
   - **Initialize with README:** ✅ Check this
   - **.gitignore:** None (for now)
   - **License:** None (for now)
4. Click **Create repository**

✅ **Checkpoint:** You should now see your new repository page.

---

## Part 2 — Explore the Repository Interface

Take 5 minutes to click around and find each of these. Write down what you find:

### The File List
- How many files are in the repo right now?
- What is the name of the file?
- What does the file contain?

### The Commit Count
- Click on the commits link (shows "1 commit")
- What information does the commit show?
- Who is listed as the author?

### The Branch Indicator
- What branch are you on?
- Click the branch dropdown — what options do you see?

### The Green "Code" Button
- Click it — what two URL options do you see?
- What's the difference between HTTPS and SSH URLs?

### The About Section (right sidebar)
- Is there a description?
- Are there any topics/tags?

---

## Part 3 — Edit the README on GitHub

GitHub lets you edit files directly in the browser.

1. Click on `README.md` in the file list
2. Click the **pencil icon** (Edit this file) in the top right
3. Replace the content with:

```markdown
# Bootcamp Practice

A repository for practicing Git and GitHub during the frontend bootcamp.

## What I'm Learning
- Git basics (init, add, commit, push)
- GitHub repository management
- Version control workflows

## About Me
[Your name here] — Frontend developer in training.
```

4. Scroll down to the **Commit changes** section
5. Write a commit message: `Update README with project description`
6. Make sure "Commit directly to the main branch" is selected
7. Click **Commit changes**

✅ **Checkpoint:** You should see the updated README on the repo page.

---

## Part 4 — Explore the Commit History

1. Click on **2 commits** (the commit count should now be 2)
2. Look at both commits:
   - What was the first commit? (The one GitHub made automatically)
   - What was the second commit? (The one you just made)
3. Click on your commit to see the diff
4. What does the green/red highlighting show?

📝 **Write down:** What's the difference between the two commits?

---

## Part 5 — Explore Repository Settings

1. Click the **Settings** tab
2. Find these sections and note what they do:
   - **General** — What can you change here?
   - **Collaborators** — What would you use this for?
   - **Pages** — What is GitHub Pages?
   - **Danger Zone** — What options are here? (Don't click anything!)

📝 **Write down:** What is GitHub Pages and how would you enable it?

---

## Part 6 — Create an Issue

Issues are used to track bugs, feature requests, and tasks.

1. Click the **Issues** tab
2. Click **New issue**
3. Title: `Add a projects section to the README`
4. Description: `The README should include a section listing projects I've built during the bootcamp.`
5. Click **Submit new issue**

✅ **Checkpoint:** You should see your issue listed in the Issues tab.
