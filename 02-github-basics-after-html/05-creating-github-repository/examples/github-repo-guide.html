Creating a GitHub Repository

From zero to a live repository — step by step

Steps to Create a New Repository

1

Go to github.com and click the "+" button → "New repository"

Or go directly to github.com/new

2

Choose a clear, descriptive name

Use lowercase letters and hyphens. No spaces. See naming examples below.

3

Add a short description (optional but recommended)

2–3 sentences about what the project does. This shows on your profile.

4

Choose Public or Private

Public = anyone can see it. Private = only you (and collaborators). Portfolio projects should be Public.

5

Initialize with a README (recommended)

Creates a README.md file automatically. You can also add a .gitignore template and a license.

6

Click "Create repository"

Your repo is live! Copy the clone URL to start working locally.

Repository Naming Conventions

✅ Good Names

  • my-portfolio
  • todo-app-vanilla-js
  • weather-dashboard
  • alex-rivera-portfolio
  • frontend-bootcamp-projects
  • restaurant-landing-page

❌ Bad Names

  • test
  • untitled
  • asdfgh
  • MyPortfolio (use hyphens)
  • my portfolio (no spaces)
  • project1

Public vs Private

SituationUse

Projects with API keys or secrets🔒 Private

🌤️ Weather Dashboard

A responsive weather app that shows current conditions and a 5-day forecast for any city. Built with vanilla HTML, CSS, and JavaScript using the OpenWeather API.

🛠️ Technologies

  • HTML5 & CSS3
  • Vanilla JavaScript (ES6+)
  • OpenWeather API

🚀 How to Run

1. Clone the repo: git clone https://github.com/username/weather-dashboard

2. Open index.html in your browser

3. Enter a city name and press Enter

GitHub Pages — Free Hosting for Static Sites

  1. Go to your repo → Settings → Pages
  2. Under "Source", select Deploy from a branch
  3. Branch: main, Folder: / (root)
  4. Click Save
  5. Wait ~1 minute, then your site is live at:
https://yourusername.github.io/repo-name

✅ Works with: HTML, CSS, JavaScript, images
❌ Does NOT work with: PHP, Python, Node.js, databases

Connect a Local Repo to GitHub

# After creating the repo on GitHub, run these commands locally:

$ git init
$ git add .
$ git commit -m "Initial commit"
$ git branch -M main
$ git remote add origin https://github.com/username/repo-name.git
$ git push -u origin main

# If your local branch is "master" but GitHub expects "main":
$ git branch -M main

License Types

MIT ⭐ Recommended

Anyone can use, copy, modify, and distribute your code. Just keep the copyright notice. Best for most projects.

Apache 2.0

Like MIT but includes patent protection. Good for larger projects.

GPL v3

Derivatives must also be open source. Good if you want to keep the ecosystem open.

No License

You keep all rights. Others technically cannot use your code without permission.
