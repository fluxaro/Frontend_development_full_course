.gitignore

Tell Git which files to never track — keep secrets safe and repos clean

What is .gitignore?

The rule: Any file or folder listed in .gitignore will never be committed to Git. It's invisible to Git — as if it doesn't exist.

Pattern Syntax

PatternWhat it matches
*.logAny file ending in .log (anywhere)
temp/The temp folder (trailing / = folder only)
**/temp/temp folder anywhere in the project
!keep.logException — do NOT ignore this file
doc/*.txt.txt files in doc folder only (not subdirs)
doc/**/*.txt.txt files in doc and all subdirectories
secret.txtExact file named secret.txt
*.envAny file ending in .env

Pattern Examples

📄 .gitignore — pattern examples
# Ignore a specific file
secret.txt

# Ignore all files with extension
*.log
*.env

# Ignore a folder (trailing slash = folder)
node_modules/
dist/

# Ignore folder anywhere in the project
**/temp/

# Exception: do NOT ignore this specific file
!important.log

# Ignore .txt files in logs/ folder only
logs/*.txt

# Ignore .txt files in logs/ and all subdirs
logs/**/*.txt

Untrack Files Already Committed

Problem: You committed node_modules/ or .env before adding .gitignore. Adding them to .gitignore now won't remove them from Git history — you need to untrack them.
Untrack files that should be ignored
# Untrack a single file (keeps it on disk)
$ git rm --cached .env

# Untrack a folder
$ git rm --cached -r node_modules/

# Then commit the removal
$ git add .gitignore
$ git commit -m "Remove tracked files that should be ignored"
$ git push

Global .gitignore — For OS Files

Set up a global gitignore
# Create a global gitignore file
$ touch ~/.gitignore_global

# Tell Git to use it
$ git config --global core.excludesfile ~/.gitignore_global

# Add OS-specific files to it (so you don't repeat in every project)
.DS_Store
Thumbs.db
.vscode/
.idea/

Common Mistakes

❌ Not adding .gitignore first

  • Create .gitignore before
  • your first commit
  • Otherwise you must
  • untrack files manually

❌ Committing node_modules

  • Can be 100MB+
  • Slows down cloning
  • npm install recreates it
  • Always ignore it

❌ Committing .env files

  • Contains API keys
  • Contains passwords
  • Public repos = exposed
  • Rotate keys immediately

⚠️ Missing trailing /

  • dist ignores files AND
  • folders named "dist"
  • dist/ ignores only
  • the folder
