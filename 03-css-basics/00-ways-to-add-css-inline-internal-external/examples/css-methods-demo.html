Three Ways to Add CSS

This page demonstrates all three methods of adding CSS to HTML.

Method 1: Inline CSS

Inline Styles

This paragraph has inline CSS applied directly via the style attribute. Notice the yellow background and gray text.

Pros: Highest specificity, no extra files
Cons: Hard to maintain, cannot reuse, mixes content with presentation

Method 2: Internal CSS

Internal Styles

This section is styled by the <style> block in the <head> of this document. The .method-section class and .method-label class are defined there.

Pros: Applies to whole page, no extra HTTP request
Cons: Cannot reuse across pages, page file gets large

Method 3: External CSS (Recommended)

External Stylesheet

In a real project, you would link an external style.css file: 

<link rel="stylesheet" href="style.css">

The CSS file contains all your styles, and every HTML page links to it. When you update the CSS file, all pages update automatically.

Pros: Reusable, browser caches it, clean separation
Cons: Extra HTTP request (negligible with HTTP/2)

The Cascade Priority

Which Wins?

  1. Inline styles — highest priority
  2. Internal styles — medium priority
  3. External styles — normal priority
  4. Browser defaults — lowest priority

When multiple rules target the same element and property, the higher priority wins.
