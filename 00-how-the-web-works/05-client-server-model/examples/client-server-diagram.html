The Client-Server Model

The client-server model describes how two computers communicate over a network. One computer (the client) makes requests, and another (the server) responds.

How It Works — Step by Step

  1. Client sends a request

    Your browser (the client) sends an HTTP request to a server asking for a resource.

  2. Request travels over the internet

    The request is broken into packets and routed through the internet to the server.

  3. Server receives and processes the request

    The server reads the request, may query a database, and prepares a response.

  4. Server sends a response

    The server sends back an HTTP response with a status code and the requested content.

  5. Client receives and displays the response

    The browser reads the HTML, CSS, and JavaScript and renders the page.

Types of Servers

Web Server
Serves static files like HTML, CSS, images. Examples: Apache, Nginx.
Application Server
Runs business logic and dynamic code. Examples: Node.js, Django, Rails.
Database Server
Stores and retrieves data. Examples: PostgreSQL, MySQL, MongoDB.
File Server
Stores and serves files. Examples: Amazon S3, Google Cloud Storage.
CDN (Content Delivery Network)
Distributes static files from servers close to the user for faster delivery.
