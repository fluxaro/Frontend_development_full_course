The Internet Journey

This page explains how data travels across the internet from your computer to a web server and back.

Step-by-Step: What Happens When You Visit a Website

Step 1 — You Type a URL

You open your browser and type https://www.example.com and press Enter.

Step 2 — Browser Checks the Cache

The browser first checks if it already has a saved copy of the page. If yes, it shows that. If no, it continues.

Step 3 — DNS Lookup

The browser asks a DNS server: "What is the IP address for example.com?" The DNS server replies with something like 93.184.216.34.

Step 4 — TCP Connection

Your browser opens a connection to the server at that IP address using the TCP protocol. This is called a "handshake."

Step 5 — HTTP Request Sent

Your browser sends an HTTP GET request asking for the page content.

Step 6 — Server Responds

The web server receives the request, finds the page, and sends back an HTTP response with the HTML content.

Step 7 — Browser Renders the Page

Your browser reads the HTML, loads any CSS and JavaScript files, and displays the finished page.

Key Terms Glossary

Internet
A global network of computers connected together, able to communicate using shared protocols.
Protocol
A set of agreed-upon rules for how computers communicate. Examples: HTTP, TCP/IP, DNS.
Packet
A small chunk of data. Large files are split into many packets, sent separately, and reassembled.
IP Address
A unique identifier for a device on a network. Example: 192.168.1.1 (IPv4) or 2001:db8::1 (IPv6).
ISP
Internet Service Provider. The company that gives you access to the internet (e.g. Comcast, AT&T).
DNS
Domain Name System. Converts domain names to IP addresses.
HTTP
HyperText Transfer Protocol. The rules for sending web pages between browsers and servers.
HTTPS
HTTP Secure. Same as HTTP but with encryption to protect your data.
Bandwidth
The maximum amount of data that can be transferred per second.
Latency
The time delay between sending a request and receiving a response.
