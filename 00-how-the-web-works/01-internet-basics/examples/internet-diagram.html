How the Internet Works

When you visit a website, your data travels through several steps before the page appears on your screen.

The Journey of a Request

  1. You (the User)

    You type a web address into your browser.

  2. Your Browser

    The browser prepares an HTTP request to send to the server.

  3. Your ISP (Internet Service Provider)

    Your internet provider routes your request toward the destination.

  4. DNS Server

    The DNS server translates the domain name (e.g. google.com) into an IP address.

  5. The Web Server

    The server receives your request and prepares a response (the HTML page).

  6. Response Travels Back

    The server sends the HTML, CSS, and JavaScript files back to your browser.

  7. Your Browser Renders the Page

    The browser reads the files and displays the finished webpage on your screen.

Key Concepts

Packets
Data is broken into small chunks called packets. Each packet travels independently and is reassembled at the destination.
Protocol
A set of rules computers agree on to communicate. HTTP, TCP/IP, and DNS are all protocols.
IP Address
A unique number assigned to every device on the internet. Like a home address for your computer.
DNS
Domain Name System. Translates human-readable names like "google.com" into IP addresses like 142.250.80.46.
Bandwidth
How much data can travel at once. More bandwidth means faster downloads.
Latency
How long it takes for a packet to travel from sender to receiver. Lower latency means a faster connection.

Analogy: The Postal System

Think of the internet like a postal system:

  • Your message is the request
  • The envelope is a packet
  • The post office is the ISP
  • The address book is DNS
  • The recipient is the web server
  • The reply letter is the HTTP response
