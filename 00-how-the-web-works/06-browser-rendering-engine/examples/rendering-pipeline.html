How the Browser Renders a Page

When your browser receives HTML from a server, it goes through 6 steps before you see anything on screen. This process is called the Critical Rendering Path.

Step 1: Parse HTML — Build the DOM

The browser reads the HTML file from top to bottom and builds a tree structure called the DOM (Document Object Model). Every HTML element becomes a node in this tree.

HTML Input:
<html>
  <body>
    <h1>Hello</h1>
    <p>World</p>
  </body>
</html>

DOM Tree:
html
└── body
    ├── h1 → "Hello"
    └── p  → "World"

Step 2: Parse CSS — Build the CSSOM

While building the DOM, the browser also parses all CSS files and builds the CSSOM (CSS Object Model). This is a tree of all CSS rules and their computed values.

Important: CSS is render-blocking. The browser will not show anything until all CSS is downloaded and parsed.

Step 3: Combine DOM + CSSOM — Build the Render Tree

The browser combines the DOM and CSSOM into a Render Tree. This tree only contains visible elements. Elements with display: none are excluded from the Render Tree.

Step 4: Layout (Reflow)

The browser calculates the exact position and size of every element on the page. This answers: "Where does each box go? How wide and tall is it?"

Any time you change an element's size or position, the browser must redo this step. This is called a reflow and it is expensive.

Step 5: Paint (Rasterize)

The browser fills in the pixels — colors, text, images, borders, shadows. This is called painting or rasterization.

Changing a color triggers a repaint but not a reflow. Repaints are cheaper than reflows.

Step 6: Composite

If the page has multiple layers (from CSS transforms, opacity, z-index), the browser composites them together in the correct order to produce the final image on screen.

CSS transform and opacity only trigger compositing — they skip Layout and Paint entirely, making them the cheapest properties to animate.

Key Terms

DOM (Document Object Model)
A tree representation of your HTML that the browser builds and uses internally.
CSSOM (CSS Object Model)
A tree representation of your CSS rules and their computed values.
Render Tree
The combined DOM + CSSOM, containing only visible elements with their styles.
Layout / Reflow
The step where the browser calculates the position and size of every element.
Paint / Rasterize
The step where the browser draws the actual pixels on screen.
Composite
The step where the browser combines multiple layers into the final image.
Render-blocking
Resources (CSS, sync JS) that prevent the browser from rendering until they are loaded.
Critical Rendering Path
The sequence of steps the browser must complete before showing the first pixel.
