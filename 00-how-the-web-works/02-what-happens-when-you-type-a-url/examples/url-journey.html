What Happens When You Type a URL?

Typing a web address and pressing Enter triggers a complex chain of events. Here is each step explained simply.

The 7 Steps

  1. You Type the URL

    Example: https://www.github.com

    The browser parses the URL into parts: protocol (https), domain (www.github.com), and path (/).

  2. DNS Lookup

    The browser asks: "What is the IP address for github.com?"

    The DNS server responds with an IP address like 140.82.121.4.

  3. TCP Connection (Three-Way Handshake)

    Your browser and the server establish a connection:

    • Browser sends: SYN (I want to connect)
    • Server replies: SYN-ACK (OK, I'm ready)
    • Browser confirms: ACK (Great, let's go)

  4. TLS Handshake (for HTTPS)

    Because the URL uses https, the browser and server agree on encryption keys. This keeps your data private.

  5. HTTP Request Sent

    The browser sends a GET request to the server asking for the page.

    GET / HTTP/1.1
Host: www.github.com
Accept: text/html

  6. Server Sends HTTP Response

    The server finds the page and sends it back with a status code and the HTML content.

    HTTP/1.1 200 OK
Content-Type: text/html

<!DOCTYPE html>
<html>...</html>

  7. Browser Renders the Page

    The browser reads the HTML, requests any linked CSS and JavaScript files, builds the page, and displays it on your screen.

Key Terms

URL
Uniform Resource Locator. The full web address including protocol, domain, and path.
DNS
Domain Name System. Translates domain names into IP addresses.
TCP
Transmission Control Protocol. Ensures data is delivered reliably and in order.
TLS
Transport Layer Security. Encrypts data between browser and server (used in HTTPS).
HTTP Request
A message from the browser asking the server for a resource.
HTTP Response
The server's reply, containing a status code and the requested content.
Rendering
The process of turning HTML, CSS, and JavaScript into a visible webpage.
