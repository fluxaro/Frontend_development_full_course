What Happens When You Type a URL — 7 Steps

This is a complete walkthrough of every step that happens between pressing Enter and seeing a webpage.

Step 1: URL Parsing

The browser breaks the URL into parts:

  • Protocol: https — how to communicate
  • Domain: www.github.com — who to talk to
  • Path: /explore — which page to get
  • Query: ?q=react — extra parameters

Step 2: DNS Resolution

The browser needs to find the IP address for the domain. It checks in this order:

  1. Browser cache (did I look this up recently?)
  2. Operating system cache
  3. Router cache
  4. ISP DNS server
  5. Root DNS servers (if all else fails)

Result: github.com → 140.82.121.4

Step 3: TCP Three-Way Handshake

Before any data is sent, a connection must be established:

  1. SYN — Browser: "I want to connect"
  2. SYN-ACK — Server: "OK, I'm ready"
  3. ACK — Browser: "Great, let's go"

Step 4: TLS Handshake (HTTPS only)

For secure connections, the browser and server agree on encryption:

  • Server sends its SSL certificate
  • Browser verifies the certificate is valid and trusted
  • Both sides agree on encryption keys
  • All future data is encrypted

Step 5: HTTP Request

The browser sends a request to the server:

GET /explore HTTP/1.1
Host: github.com
Accept: text/html,application/xhtml+xml
Accept-Language: en-US
Connection: keep-alive

Step 6: Server Response

The server processes the request and sends back a response:

HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Content-Length: 45231

<!DOCTYPE html>
<html lang="en">
  ...
</html>

Step 7: Browser Rendering

The browser turns the raw files into a visible page:

  1. Parse HTML → build the DOM (Document Object Model)
  2. Parse CSS → build the CSSOM (CSS Object Model)
  3. Combine DOM + CSSOM → Render Tree
  4. Layout — calculate where everything goes on screen
  5. Paint — draw pixels on screen
  6. Composite — layer everything together
