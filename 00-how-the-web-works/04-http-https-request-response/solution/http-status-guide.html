HTTP Methods

GET — Retrieve Data

Used to request data from a server. Does not change anything on the server. Safe to call multiple times.

GET /api/users HTTP/1.1
Host: api.example.com

POST — Create Data

Used to send data to the server to create a new resource. Calling it twice creates two resources.

POST /api/users HTTP/1.1
Host: api.example.com
Content-Type: application/json

{"name": "Alice", "email": "alice@example.com"}

PUT — Replace Data

Replaces an entire resource with new data. If the resource doesn't exist, it may create it.

PUT /api/users/1 HTTP/1.1
Content-Type: application/json

{"name": "Alice Smith", "email": "alice@example.com"}

PATCH — Partially Update Data

Updates only the specified fields of a resource. More efficient than PUT when changing one field.

PATCH /api/users/1 HTTP/1.1
Content-Type: application/json

{"email": "newemail@example.com"}

DELETE — Remove Data

Removes a resource from the server. Calling it again on the same resource is safe (idempotent).

DELETE /api/users/1 HTTP/1.1
Host: api.example.com

HTTP Status Codes

1xx — Informational

CodeNameWhen You See It
100ContinueServer received request headers, client should proceed
101Switching ProtocolsServer is switching to WebSocket protocol

2xx — Success

CodeNameWhen You See It
200OKStandard success response
201CreatedAfter a successful POST that created a resource
204No ContentSuccess with no response body (after DELETE)
206Partial ContentRange request for video streaming or large files

3xx — Redirection

CodeNameWhen You See It
301Moved PermanentlyURL has permanently changed (SEO-friendly redirect)
302FoundTemporary redirect
304Not ModifiedBrowser should use its cached version
307Temporary RedirectRedirect but keep the same HTTP method
308Permanent RedirectLike 301 but keep the same HTTP method

4xx — Client Errors

CodeNameWhen You See It
400Bad RequestMalformed request, invalid JSON, missing required fields
401UnauthorizedNot logged in, missing or invalid token
403ForbiddenLogged in but not allowed to access this resource
404Not FoundResource doesn't exist at this URL
405Method Not AllowedTried to POST to a GET-only endpoint
409ConflictResource already exists (duplicate email on signup)
422Unprocessable EntityRequest is valid but data fails validation
429Too Many RequestsRate limit exceeded — slow down

5xx — Server Errors

CodeNameWhen You See It
500Internal Server ErrorUnhandled exception or crash on the server
501Not ImplementedServer doesn't support this feature
502Bad GatewayProxy received invalid response from upstream
503Service UnavailableServer is down for maintenance or overloaded
504Gateway TimeoutUpstream server didn't respond in time
