# Assignment — Web Developer Roles Guide

## What You Are Building

Create an HTML page called `web-roles-guide.html` that serves as a complete guide to the three web development roles. Write it as if you are explaining it to a friend who has never heard of web development.

No CSS. No JavaScript. Pure HTML structure only.

---

## Requirements

### 1. Introduction
An `<h1>` and a paragraph explaining that web development has different specializations and why understanding them matters.

### 2. Three Role Sections
For each role (Frontend, Backend, Fullstack), create a `<section>` with:
- `<h2>` with the role name
- `<p>` describing what the role does in plain language
- A real-world analogy (restaurant, building, etc.)
- `<ul>` of key technologies
- `<ul>` of common job titles
- `<p>` describing a typical day in that role

### 3. Technology Comparison Table
A `<table>` comparing the three roles across: Languages, Frameworks, Databases, Tools, Runs where.

### 4. Common Tech Stacks Section
An `<h2>` called "Common Tech Stacks" with a `<table>` showing at least 4 popular stacks (MERN, MEAN, T3, LAMP) and what each letter stands for.

### 5. Learning Path Section
An `<h2>` called "Recommended Learning Path" with an `<ol>` showing the order to learn technologies for a frontend developer.

### 6. Resources Section
An `<h2>` called "Where to Learn More" with a `<ul>` of links to relevant resources (MDN, freeCodeCamp, The Odin Project, etc.).

---

## What Good Looks Like

- Written in plain language a beginner can understand
- All tables have `<thead>`, `<tbody>`, and `<caption>`
- Proper heading hierarchy throughout
- Links in the resources section use `target="_blank"` and `rel="noopener noreferrer"`

Save as `web-roles-guide.html` and submit via GitHub.
