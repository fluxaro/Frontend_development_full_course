# Class Work — Frontend vs Backend vs Fullstack

## What You Will Do

Research the three web development roles and build an HTML comparison page that explains each one clearly.

**Time:** 30 minutes

---

## Activity 1: Research (10 minutes)

Look up the following on Google or LinkedIn Jobs:

1. Search "frontend developer job" — what skills are listed most often?
2. Search "backend developer job" — what skills are listed most often?
3. Search "fullstack developer job" — what skills are listed most often?

Write down:
- 3 skills that appear in almost every frontend job listing
- 3 skills that appear in almost every backend job listing
- What is the most common frontend framework mentioned?
- What is the most common backend language mentioned?

---

## Activity 2: Build a Comparison Page (20 minutes)

Create a file called `web-roles-comparison.html`.

Using only HTML, build a page that includes:

### Required Sections

1. **`<h1>`** — "Web Development Roles"

2. **Three role sections** — one each for Frontend, Backend, Fullstack. Each section must have:
   - `<h2>` with the role name
   - `<p>` describing what the role does
   - `<ul>` listing the key technologies
   - `<p>` with a real-world analogy

3. **A comparison `<table>`** with columns: Aspect | Frontend | Backend | Fullstack
   - Include rows for: What they build, Languages, Frameworks, Runs where, Visible to users

4. **A "Which Should I Learn First?" section** with your recommendation and reasoning

5. **A "Job Titles" section** with a `<dl>` listing common job titles for each role

---

## Checklist

- [ ] All three roles are explained clearly
- [ ] Each role has a technology list
- [ ] Comparison table is complete and accurate
- [ ] Page uses proper semantic HTML
- [ ] Page opens in browser with no errors
